
# **CHECK LIST — TRABAJO FINAL INTEGRADOR (Diseño Web Nativo y Accesible)**  
_Plataforma educativa — HTML + CSS + JS_

---

## **1. Estructura General del Sitio (HTML)**  
- Estrutura por carpetas
- Semántica estructural — Uso correcto de: 
    `<header>`, 
    `<nav>`, 
    `<main>`,
    `<section>`, 
    `<article>`, 
    `<aside>`, 
    `<footer>`  
- Cuatro páginas HTML — index + 3 internas  
    index.HTML
    sobrenosotros.html
    nuestroscursos.html
    cursocolores.html
    contacto.html
- Meta viewport — incluida y validada  
- Validación W3C — sin errores  
- Footer institucional — logos de certificación HTML/CSS  
- Algoritmo de esquematizado — jerarquía clara de `<h1>` a `<h6>`

---

## **2. Diseño y Maquetación (CSS)**  
- Separación total HTML/CSS — sin estilos inline  
- Layout con Grid  
  - Index sin `<aside>`  
  - Páginas internas con `<aside>` lateral  
- Flexbox interno — 
    navegación, 
    cards, 
    bloques internos  
- Pseudoclases accesibles —
 `:hover`, 
 `:focus`, 
 `:active`  
- Variables CSS — paleta + modo oscuro  
- Media Queries (optativo) — adaptación móvil

---

## **3. Funcionalidad Dinámica (JavaScript)**  
- Modo claro/oscuro — toggle + manipulación del DOM  
- Clases dinámicas — agregar/quitar clases  
- Código limpio — comentarios y orden

---

## **4. Página Interna 1 — Texto y Tipografía**  
- Etiquetas semánticas de texto — `<strong>`, `<em>`, `<mark>`, `<cite>`  
- Tipografía accesible — interlineado, espaciado, fuentes seguras  
- Contraste AAA — según WCAG

---

## **5. Página Interna 2 — Formulario Accesible**  
- Estructura semántica — `<form>`, `<label for="">`, `<fieldset>`, `<legend>`  
- Validación nativa HTML5 — `required`, `pattern`, `minlength`, `type="email"`  
- Accesibilidad ARIA — `aria-labelledby`, `aria-describedby`  
- Mensajes de error accesibles

---

## **6. Página Interna 3 — Galería de Imágenes en Cards**  
- Uso de `<figure>` y `<figcaption>`  
- Atributo alt descriptivo  
- Efecto visual interactivo — `transform` + `transition`  
- Cards accesibles

---

## **7. Accesibilidad Global del Sitio**  
- Navegación por teclado  
- Foco visible  
- Jerarquía de información  
- Roles ARIA solo cuando sean necesarios

## **8. Optimización y Calidad del Código**  
- Código limpio y ordenado  
- Comentarios útiles  
- Archivos organizados — `/css`, `/js`, `/img`, `/pages`  
- Certificaciones W3C en footer

## **9. Declaración de Uso de IA**  
- Declaración incluida  
- Redacción clara y ética

- Proyecto completo en carpeta  
- Archivos vinculados correctamente  
- Pruebas en distintos navegadores
